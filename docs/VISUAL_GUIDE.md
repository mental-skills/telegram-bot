# Визуальная инструкция Mental Skills для Codex

## 1. Источники истины

Используй файлы в таком порядке приоритета:

1. `assets/reference/brand_illustration_style.png` — герои, свет, движение, атмосфера спортивного пространства.
2. `assets/reference/brand_graphic_elements.png` — линии, рамки, паттерны, акценты и композиционная система.
3. `assets/runtime/brand_logo_horizontal.png` — точное начертание и цветные нейронные акценты логотипа.
4. `assets/brand_tokens.json` — машинно-читаемые цвета и правила.
5. `assets/visual_usage_map.json` — соответствие ролей экрана и готовых карточек.
6. `assets/asset_manifest.json` — физические пути, допустимые роли, alt-тексты, размеры и контрольные суммы.

## 2. Что должен делать обычный Telegram-бот

- Отправлять готовую фирменную карточку только на ключевом смысловом этапе.
- Всегда сохранять полный текст и кнопки независимо от картинки.
- Не пытаться перекрашивать системные кнопки Telegram.
- Не создавать изображения на лету и не помещать варианты ответа внутрь картинки.
- При длинном тексте отправлять фото отдельно, затем текст с inline-кнопками.
- При ошибке файла переходить в text-only режим без изменения текущего узла.

## 3. Где использовать изображения

- `/start` — `brand_logo_horizontal`.
- Вступление в ситуацию — `ui_dialogue_choice`.
- Главная мысль — `ui_insight`.
- Практический совет — `ui_practice_task`.
- Ритуал переключения — `ui_neural_focus`.
- Домашнее упражнение — `ui_calendar`.
- Завершение ситуации — `ui_achievement`.
- Словесный итог полного модуля — `ui_rating`.
- Библиотека инструментов — `ui_tool_cards`.

## 4. Что запрещено

- Отправлять пользователю брендборды из `assets/reference/`.
- Растягивать или обрезать логотип так, что меняются пропорции.
- Перерисовывать предоставленные изображения без отдельного согласования.
- Делать нейронные цвета основным разноцветным фоном.
- Использовать проценты, диаграммы или шкалы как психологическую диагностику.
- Ставить картинку на каждый узел и замедлять прохождение.

## 5. Проверка перед сдачей

- Каждый runtime-ассет существует и читается.
- Все `asset_id` из сценария есть в manifest.
- Design-reference нельзя получить через Telegram renderer.
- Текстовый fallback проходит те же переходы и сохраняет прогресс.
- Экраны проверены на мобильном Telegram при светлой и тёмной системной теме.
- Изображение не содержит динамического текста конкретной ситуации.
