# Mental Skills — брендированный пакет для Codex, версия 2

Пакет предназначен для разработки MVP Telegram-бота — образовательного тренажёра для родителей юных футболистов — в фирменной стилистике Mental Skills.

## Состав

### Документы

- `docs/Mental_Skills_TZ_Telegram_Bot_MVP_branded_v2.docx` — обновлённое техническое задание с визуальной системой и картой ассетов.
- `docs/CODEX_PROMPT_BRANDED_V2.md` — готовое первое задание для Codex.
- `docs/Mental_Skills_football_parent_trainer_revised.docx` — утверждённая методическая база семи сценариев.

### Контент

- `content/PREMATCH_INSTRUCTIONS_02.json` — сценарий №2 с подключёнными визуальными карточками.
- `content/scenario.schema.json` — JSON Schema 1.2 с поддержкой поля `media`.

### Визуальные материалы

- `assets/asset_manifest.json` — единый каталог ассетов и их назначений.
- `assets/reference/` — брендборды для разработчика; не отправляются в Telegram.
- `assets/runtime/` — готовые изображения, которые бот может отправлять пользователю.


### Дополнительные визуальные спецификации

- `docs/VISUAL_GUIDE.md` — правила применения последних приложенных брендбордов и готовых карточек.
- `assets/brand_tokens.json` — палитра и визуальные токены.
- `assets/visual_usage_map.json` — карта ролей экранов.
- `assets/reference/runtime_assets_contact_sheet.png` — обзор runtime-ассетов с идентификаторами.

## Обязательная логика

- Бот остаётся полноценным и понятным без изображений.
- Визуальная карточка используется на смысловом этапе, а не в каждом сообщении.
- Системные кнопки Telegram не перекрашиваются.
- Два брендборда используются как эталон, но не как пользовательские экраны.
- Предоставленные изображения нельзя заменять или перерисовывать без согласования.

## Рекомендуемый порядок

1. Создать новый Git-репозиторий.
2. Перенести каталоги `docs/`, `content/` и `assets/` без изменения относительных путей.
3. Передать Codex текст из `docs/CODEX_PROMPT_BRANDED_V2.md`.
4. На первой итерации реализовать только сценарий №2 и универсальный `ScenarioEngine`.
5. Проверить переходы, сохранение прогресса, три возрастные версии и media fallback.
6. Проверить реальные экраны на iOS, Android и Desktop Telegram.
7. После приёмки механики преобразовать остальные шесть сценариев в тот же JSON-формат.

## Важный принцип

Бот является образовательным тренажёром, а не психологической диагностикой. Он не показывает проценты, диагнозы или типы личности.


## Изменения версии 2

- В качестве источников истины используются последние приложенные пользователем брендборды и горизонтальный логотип.
- Добавлены контрольные суммы, размеры и форматы всех ассетов.
- Добавлены `brand_tokens.json`, `visual_usage_map.json` и отдельная визуальная инструкция.
- JSON Schema обновлена до версии 1.2.
- Поле `media` поддерживает `presentation` и `caption_strategy`.
